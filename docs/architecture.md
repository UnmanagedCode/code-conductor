## Stack
- Node 22+ (`node:test`, top-level await, `crypto.randomUUID`).
- `express` (REST + static) + `ws` (WebSocket).
- Vanilla HTML/CSS/JS in `public/`, no build step.
- `happy-dom` (dev-only) for DOM-backed rendering tests.
- No DB ŌĆö projects in `<projectsRoot>/`, sessions in `~/.claude/projects/<encoded-cwd>/*.jsonl` (the jsonl root is overridable via `CLAUDE_PROJECTS_ROOT`).

## Component layout
- **server.js** ŌĆö Express + ws boot, mounts `/api` ŌåÆ `/mcp` ŌåÆ `/plugins` (reverse proxy) ŌåÆ static, binds `127.0.0.1:8787`. The WebSocketServer is `noServer: true` with a manual `server.on('upgrade')` dispatcher: `/ws` ŌåÆ wsHub, `/plugins/<id>/ŌĆ”` ŌåÆ plugin proxy raw-socket pipe, anything else destroyed. (With `noServer` the ws lib never attaches to the http server, so the old EADDRINUSE error-forwarding guard around `listenWithRetry` is gone ŌĆö `tests/eaddrinuse-retry.test.mjs` covers the retry path.) Creates the plugin host and, after listen, feeds it the bound port + fires its adopt-on-boot init. Sets `process.title = 'code-conductor'` on entry so `pkill -f code-conductor` targets it precisely and `pkill -f server.js` misses it. Launched via `conductor.sh` (`npm start`).
- **src/instances.js** ŌĆö Instance class, InstanceManager, capped event ring (`EventLog`: drop-oldest at `ORCH_EVENT_RING_CAP`, default 2000; `_seq` stamped at push and never renumbered; `trimmedBefore` = first retained `_seq`; trim snaps the surviving head to an outer `user_echo` turn boundary; `snapshotTail()` serves the Ōēż`ORCH_SNAPSHOT_TAIL` (default 500) trailing slice for WS subscribe), absolute `userIndex` stamping on outer user_echoes (the rewind/fork anchor ŌĆö immune to trimming), control_request round-trip, mode validation. **Resume idempotency:** `create()` is a thin synchronous wrapper over `_doCreate()` that guards duplicate resumes of one sessionId ŌĆö an already-live session (`i.proc`) throws 409 `already attached`, and a still-in-flight resume (tracked in `this._resuming: sessionIdŌåÆPromise`) is **coalesced** (the second concurrent caller gets the same restored instance, one subprocess). The checks run in the wrapper's synchronous prefix (before any `await`), closing the window `_doCreate` opens between the guard and `spawn()` attaching `.proc`; the map entry is released when the create settles (success OR failure). This makes the restart anchor auto-resume racing the manifest restore ŌĆö and manual stop+resume ŌĆö safe regardless of caller.
- **src/idleSubscriptions.js** ŌĆö `IdleSubscriptionHub`, the idle-subscription graph extracted from `InstanceManager` as a composed collaborator. Owns the `targetSessionId ŌåÆ Map<callerSessionId,{timerId}>` map; one-shot consume-on-fire + an **always-armed** `timeoutMs` watchdog (explicit finite value, else `DEFAULT_SUBSCRIBE_TIMEOUT_MS` = 30 min via `ORCH_SUBSCRIBE_TIMEOUT_MS`; `.unref()`'d) + the `turn_end` wake-stub text. `onTurnEnd` **defers** delivery while the target's `activeAgentTaskCount > 0` (keeping the subscription + watchdog armed), so a wake fires only once the worker's turn ends with no live background subagents; a subagent that never finishes is caught by the watchdog. Cross-instance lookups + `subscription_changed` emission route back through the manager. `InstanceManager` delegates the `subscribeIdle`/`unsubscribeIdle`/`isIdleCaller`/ŌĆ” surface and exposes the live `_idleSubscribers` getter. `deliver()` picks one of three stubs (`_plainStub`/`_buildFoldedStub`): a real `turn_end` to an already-idle caller **folds** the worker's recent output inline (reusing `buildRecentMessages` from `mcp/handlers.js` + `flattenPayload` from `mcp/content.js`, tagged with the `public/wakeCallback.js` marker for the collapsible UI bubble); the timeout-watchdog and live mid-turn steering paths keep the plain pointer stub (a mid-turn wake is injected live into the caller's running turn). No import cycle ŌĆö nothing in `handlers.js`'s import graph reaches back to `instances.js`.
- **src/overageResume.js** ŌĆö `OverageResumeController`, the overage auto-resume timer machine extracted from `InstanceManager` as a composed collaborator. Owns the `sessionId ŌåÆ fireAtMs` (wall-clock epoch-ms **deadline**) map and `AUTO_RESUME_TEXT` (re-exported from `instances.js` for back-compat); the per-Instance overage flags (`autoResumeAt`/`autoStoppedForOverage`/`_overageHandled`/`_overageResetsAt`) stay on the Instance. Firing is driven by a **single shared wall-clock sweep interval** (`ORCH_OVERAGE_RESUME_SWEEP_MS`, default 30 s, `unref()`'d ŌĆö started on first arm, stopped when the map empties), **not** one `setTimeout` per session: each tick re-reads `Date.now()` and fires every deadline that is now due, so a deadline that lapsed while the process was suspended (Android Doze / Termux backgrounding freezes the libuv monotonic clock a `setTimeout` rides) fires on the first tick after wake. `resetsAt`-based scheduling, deadlines in-memory but persisted in the resume manifest (survive a graceful restart), never kills/respawns. **Usage-verified fire:** a due deadline does NOT resume on our clock alone ŌĆö the tick does ONE coalesced `fetchUsage()` (`getAccountUsage`, injectable seam; its 180 s cache + backoff absorb the pressure ŌĆö one fetch shared across all sessions due that cycle, guarded by `_ticking`) and only resumes when `usageOverThreshold(usage) === false` (five-hour util `< 100` and under any enabled threshold ŌĆö the SAME evaluator the poll trip uses). Still-over ŌåÆ `_reschedule` to `max(resetsAt, now + ORCH_OVERAGE_RECHECK_MS)` (also pushes the manager's global `_overageResetsAt` forward so the lockout gate's future-resetsAt rail stays satisfied while parked), naturally polling ~every minute until the account clears ŌĆö which is what kills the boundary re-trip (no resume turn is ever sent while still throttled). Can't-confirm (null/backoff/malformed) reschedules too, up to `FAIL_OPEN_AFTER`=5 consecutive failures, then fails open and resumes so a persistently-down usage API can't park a session forever. `arm()` no longer dead-ends on a missing/past `resetsAt` ŌĆö it schedules the first check at `now + recheck`; **`auto_resume_skipped`** now fires ONLY for the genuine "process gone" case in `run()`. `armRestored` (restart) re-inserts its persisted deadline as-is and inherits the same usage-verified fire. `InstanceManager` delegates the `_armAutoResume`/`_cancelAutoResume`/ŌĆ” surface (also called from the temp-session exit branch so a disposed temp session never leaves an orphan deadline) and exposes the live `_autoResumeTimers` getter. **Message queueing:** while a session is auto-stopped-and-armed, a non-internal `Instance.prompt()` is intercepted **before** `emit('user_prompt')` and pushed onto `inst._overageQueue` (`{text, attachments, ts}`) with an `overage_message_queued` UI event ŌĆö it does NOT resume or cancel. `run()` flushes on fire: it snapshot-empties the queue, calls `cancel(sid)` FIRST (clears `autoStoppedForOverage`/`autoResumeAt` so the follow-on non-internal resume send isn't re-queued ŌĆö `cancel` is idempotent so the send's own `user_promptŌåÆcancel` is a no-op), emits an `auto_resume` system line, then sends ONE combined `prompt()` (`buildCombinedResumeText` = `AUTO_RESUME_TEXT` + numbered list, with all queued attachments concatenated). `cancel()` also clears `_overageQueue` (kill/remove/temp-exit/shutdown cleanup). `summary()` exposes `queuedCount` (rides every status emit ŌåÆ header/sidebar badges append `┬Ę N queued`). `fireNow()` is the sweep-internal + test seam and is also the primitive `InstanceManager.reevaluateOverageResumes()` drives (one `fireNow` per parked sessionId, snapshotting `timers.keys()` first since `fireNow` synchronously deletes its own entry) ŌĆö called after a Settings ŌåÆ Models Apply that raises/disables the threshold, so a session parked under the OLD bar resolves against the new one immediately instead of waiting out its deadline. There is still **no** early-resume UI/button ŌĆö this is a settings-save side effect, not a user-facing control.
- **src/usageOverageMonitor.js** ŌĆö `UsageOverageMonitor`, the server-side usage poller composed by `InstanceManager` as a second, equal-footing overage-trip source alongside the stream `rate_limit_event`. On an `ORCH_USAGE_POLL_MS` interval (default 180 s; `unref()`'d), `_tick()` is heavily gated (skip if `_overageActive`, threshold disabled, `getOnOverageAction()==='none'`, or nothing mid-turn ŌĆö so it doesn't fetch needlessly), then calls `getAccountUsage()` (injectable as `fetchUsage` for tests; a throw or null is swallowed ŌåÆ no false-trip) and, when the **five-hour** window's `utilization` (a 0ŌĆō100 percent) `>= overageThresholdPct`, calls `manager._handleOverageTrip(null, { resetsAt })` ŌĆö the *same* seam the stream trip lands in. It writes no parallel stop logic and injects no prompts (routing's `internal:true` prompts are inherited), so dedup (`_overageActive`), conductor/conducted/plain routing, and resume all behave identically regardless of source. `start()`/`stop()` are idempotent; the server starts it after `listen()`, and both `shutdown()` and `shutdownForResumeSync()` stop it. The point: the stream event only reports near Anthropic's own ~90% mark, so a *low* threshold (e.g. 25%) is invisible to it ŌĆö only the poll catches it. `parseResetEpochSecs` is exported from `instances.js` and reused here to turn the bucket's ISO `resets_at` into epoch seconds. `forceTick()` is a thin public passthrough to `_tick()` ŌĆö the same gates apply ŌĆö used by the Settings ŌåÆ Models Apply handler (`src/routes.js`) so a *lower* threshold that's already crossed trips a mid-turn session immediately instead of waiting up to `ORCH_USAGE_POLL_MS` for the next scheduled tick.
- **Overage trip detection + central routing** (in `src/instances.js`) ŌĆö the *decision* of what to do on an overage trip moved off the per-Instance handler (which can't reach the idle-subscription graph) up to `InstanceManager`. Two sources reach `_handleOverageTrip`: the per-Instance stream path below, and the `UsageOverageMonitor` poll (above) ŌĆö deduped by the global `_overageActive` one-shot so they never double-trip (whichever observes the crossing first wins). Flow: `Instance._isOverageTrip(data)` (hard `isUsingOverage` flag OR, when `getOverageThreshold().enabled`, `utilization >= pct/100` for any window) ŌåÆ `Instance` emits an `overage` signal (throttled per run by `_overageHandled`) ŌåÆ `InstanceManager._handleOverageTrip` holds a global one-shot `_overageActive`/`_overageResetsAt`/`_overageClearTimer`, returns early when `getOnOverageAction()==='none'`, else `_routeOverageStop` (steer-first: `_steerConductor` via `windDown`/`prompt` with `overageConductorSteerText({hasWorkers})` for in-control conductors of conducted workers, `_directOverageStop` via `interrupt()` for everything else still mid-turn ŌĆö **except** the Conduct orchestrator (`project === '.conduct'`), which is ALWAYS steered gracefully even with no in-control workers to halt, `hasWorkers:false` dropping the worker-halting sentence so it isn't sent chasing phantom workers; owner resolved by `_ownerConductor` = `byId.get(worker.callerInstanceId)`), and arms `_armOverageClear(resetsAt)`. **Global release is unified with the per-session rechecks:** the clear timer's callback is now `_maybeReleaseOverageLock` (a BACKSTOP for the never-armed case), which clears the global flag ONLY when nothing is parked (`_overageResume.timers.size === 0`) AND no live instance is still `autoStoppedForOverage` (a session soft-interrupted but not yet arrived at idle will arm imminently ŌĆö closes the past-resetsAt backstop race). The resume controller calls `_maybeReleaseOverageLock` after every deadline-removing outcome (verified resume, fail-open resume, process-gone teardown), so the lockout lifts exactly when the last parked session resolves ŌĆö the release and the resumes can't disagree because they're the same usage-verified sweep. `_clearOverage` (backstop fire, that release path, or manual takeover of an `autoStoppedForOverage` session via the `user_prompt` handler) releases the flag and resets every instance's `_overageHandled`. For `stop-resume`, **both** conductor-steer branches arm resume ŌĆö mid-turn (`windDown`, flags set before) and idle+subscribed (`prompt`, flags set *after* the call since `prompt()` emits `user_prompt` synchronously) ŌĆö alongside `_directOverageStop`; the timer arms on the steered/stopped session's next statusŌåÆidle. Resume timing keys off the five-hour `resetsAt` (read camelCase epoch from the live event by `parseResetEpochSecs`), **not** the far-future `overageResetsAt`. Orchestrator-injected prompts ŌĆö the idle-subscription wake stub and the conductor overage steer ŌĆö pass `prompt(..., {internal:true})` so their `user_prompt` does **not** cancel a pending resume (an overage-stopped conductor still gets woken when its worker finishes); only a genuine user/MCP prompt, or the auto-resume's own non-internal fire, runs `_cancelAutoResume`.

- **src/conductedSessions.js** ŌĆö sidecar store (`<store>/conducted-sessions.json`) of sessionIds spawned via MCP `spawn_instance` (the worker agents an orchestrator conducts). `loadAll()`/`isConducted(sid)`/`markConducted(sid)`/`unmarkConducted(sid)`, atomic write + serialised write-chain, mirroring `sessionTitles.js`. The durable half of the conducted axis.
- **session sidecar stores** ŌĆö sibling `<store>` JSON sidecars sharing the same atomic-write + advisory-lock shape as `conductedSessions.js`: `sessionTitles.js` (`{sessionId: customLabel}` ŌĆö `getTitle`/`setTitle`/`deleteTitle`), `archivedSessions.js` (archived-sid set ŌĆö `isArchived`/`markArchived`/`unmarkArchived`), `tempSessions.js` (durable temp-session markers that survive SIGKILL so a temp stays resumable across the resume-after-restart path). `storeLock.js` provides the cross-process advisory lock (`withLock()`, `O_EXCL` create) they serialise on: a held lock is reclaimed **only when the owner PID is dead** ŌĆö a live owner is never evicted on age, no matter how slow. This is load-bearing on Termux: post-restart CPU throttle (~10ŌĆō75├Ś) can starve a live holder for many seconds mid-write, and the old-server/new-server overlap during a restart is the only window with cross-process contention; the previous 10 s age-based eviction reclaimed the live holder there, letting two writers run the read-modify-write concurrently and drop each other's entries (a lost update) ŌĆö which silently un-archived a subset of sessions on every restart. A slow holder is now waited out via bounded retries (a truly wedged holder throws on acquire rather than corrupting), and release is **ownership-token-checked** so a process can't delete a successor's lock. `tempCleanup.js` writes/sweeps `<store>/pending-temp-cleanup.json` to GC temp jsonl orphaned after exit.

Health is passive (child exit events, proxy `reportUpstreamFailure`, on-demand `status()` probe) ŌĆö no periodic poller (battery). Init is lazy + memoized and keyed to `projectsRoot()` (test roots swap).

## Conventions

Load-bearing rules this codebase follows ŌĆö keep new work inside them so the project stays manageable as it grows.

1. **YAGNI ŌĆö build for the requirement in front of you, not the one you imagine.** Implement only what a current, concrete caller or test needs; don't add speculative abstractions, config knobs, or extension points "for later" ŌĆö they add surface area without a real user, and the eventual real requirement rarely matches the guess. If code stops being exercised by a real caller or test, delete it rather than keeping it "just in case" or "to trim later" ŌĆö dead code rots silently and misleads the next reader into thinking it's load-bearing.
2. **Frontend: `installX({ŌĆ”})` modules; `app.js` is wiring-only.** Feature logic lives in a `public/` module exposing an `installX({ŌĆ”getters,ŌĆ”callbacks})` factory (or a class for a stateful widget). `app.js` only builds `state`/the DOM map, calls each `installX` once, holds the returned handles, and wires `wsRouter` last. Modules read live state through injected getters (`getActiveId: () => state.activeId`) ŌĆö they never reach into globals. Don't grow `app.js` with feature logic; add a module and wire it.
3. **No god-modules ŌĆö extract composed collaborators.** When a module takes on a second responsibility, lift it into a collaborator the owner *composes*, keeping the public surface stable via delegation. Cf. `InstanceManager` ŌåÆ `IdleSubscriptionHub` + `OverageResumeController`; `mcp/handlers.js` ŌåÆ `diffPaging` + `messageReconstruction` + `content`; whisper/tts install ŌåÆ the `makeInstallRunner` factory.
4. **One service layer for REST + MCP ŌĆö don't reimplement per surface.** Git / worktree / diff / session logic lives once in `src/worktrees.js` (+ siblings) and is imported by **both** `src/routes.js` and `src/mcp/handlers.js` (shared: `isGitRepo`, `mergeWorktreeIntoParent`, `syncWorktree`, `getWorktree`, `getWorktreeMergeStatus`, `buildRebasePrompt`, `removeWorktree`, ŌĆ”). A new git/worktree op goes in the service layer and is called from both surfaces ŌĆö never duplicated.
5. **Single-source-of-truth catalogs, shipped to the client ŌĆö never client literals.** Curated catalogs (`modelVersions.js`, `whisperModels.js`, `ttsModels.js`) own the authoritative list + allow-list + policy server-side; the client *fetches* them (`/api/settings/models` ŌåÆ `public/models.js`) instead of hardcoding the canonical set. (`public/models.js` keeps a `DEFAULT_VERSIONS` first-paint fallback ŌĆö a fallback, not a second source of truth.)
6. **Cross-surface magic strings get one owner module.** A canonical string shared across surfaces lives in a single module. Cf. `planApproval.js` ŌĆö the approve/reject prompts shared by the UI buttons, the server-side auto-approve, and the MCP `approve_plan`/`reject_plan` tools; `public/userQuestionAnswers.js` ŌĆö the AskUserQuestion answer formatter shared by the UI question card and the MCP `answer_question` tool (a DOM-free module under `public/` that the server imports directly, avoiding a mirror); `public/wakeCallback.js` ŌĆö the idle wake-stub marker (`WAKE_CALLBACK_MARKER`/`WAKE_BODY_SEP`) shared by the server (build) and the client (parse/render); `SOFT_INTERRUPT_MARKER` in `parser.js`.

On **resume**: `loadHistory(sessionId)` runs before flipping to `idle` ŌĆö replays jsonl into UI events and emits a `system/history_replayed` divider. A history longer than the ring cap trims during the replay itself; the evicted prefix stays reachable via `GET /api/instances/:id/events`. On **turn end** or **mode change**: append `{"type":"last-prompt", ŌĆ”}` + `{"type":"permission-mode", ŌĆ”}` lines so `claude --resume`'s interactive picker can discover the session.

**`displayStatus` overlay (additive, not a 6th state).** The diagram above is the real `status` enum ŌĆö untouched. A backgrounded `Agent` tool call's tool_result resolves immediately (`isAsync:true`), so `turn_end` (ŌåÆ `idle`) can fire while the CLI's own `system/task_started`/`task_updated` task-tracking is still open for that call. `Instance._activeAgentTasks` (a `Map<task_id, tool_use_id>`) tracks this: incremented on `task_started`, decremented on a terminal `task_updated` (`patch.status` one of `completed|failed|cancelled|error` ŌĆö an unrecognized status is treated as non-terminal on purpose, to avoid flipping back to idle early) or on `task_notification` (belt-and-suspenders). `summary().displayStatus` reads `'running'` instead of `'idle'` while the map is non-empty; every other `status` value passes through unchanged. Reset on (re)spawn. `Instance.activeAgentTaskCount` exposes the live map size. `IdleSubscriptionHub.onTurnEnd` **gates on `activeAgentTaskCount`**: a `turn_end` while the map is non-empty **defers** the idle wake (the subscription + its watchdog stay armed) until the follow-up `turn_end` that fires once the last subagent completes and re-invokes the worker ŌĆö so a wake means "the worker AND all its subagents finished." Every status-gated frontend control still keys off the real `status`; the cosmetic surfaces (sidebar dot, sub-agents strip, header chip) read `displayStatus`.

## Migrations
`migrations/` holds idempotent migration scripts run automatically on boot (entrypoint `migrations/index.mjs`). Each self-checks "already applied?" and is a fast no-op in steady state; a script that throws **aborts the boot**. See [`migrations/migrations.md`](../migrations/migrations.md) for the listing and conventions for adding new ones.

## Testing
All tests via `node tests/run.mjs` (programmatic node:test runner ŌĆö Termux's glibc-runner wrapper hoists leading `--flags` into `NODE_OPTIONS` and refuses `--test`).

Test **files run in parallel** (node:test `isolation:'process'` ŌĆö one child process per file). The suite is isolation-safe: `bootServer` binds an ephemeral port (`listen(0)`) and `mkdtemp`s a unique home per server, so files never contend over ports or paths. Concurrency defaults to `min(4, cores/2)` ŌĆö capped low on purpose because each file boots its own express+ws and spawns fake-claude subprocesses, so oversubscribing (e.g. one runner per core) is *slower* and risks tripping the timing-sensitive waits (control-request 5s, `waitFor` 4s). Override with `TEST_CONCURRENCY=<n>` (`1` = fully serial). Measured ~79s serial ŌåÆ ~29s at the default on an 8-core device.

Default suite uses **fake-claude** (`tests/fake-claude.mjs`) via `CLAUDE_BIN`: silent until first stdin message, auto-acks `control_request`s, emits canned events from `FAKE_CLAUDE_SCENARIO`, matches `control_response`s so scenarios can branch on allow/deny. `FAKE_CLAUDE_ARGV_DUMP=<path>` captures launch argv.

The subagent-aware idle-wake deferral (`IdleSubscriptionHub.onTurnEnd` gating on `activeAgentTaskCount`) is covered by `tests/idle-subagent-defer.test.mjs` (unit-style, injected fake instances driving `manager.emit('event', turn_end)` with a settable `activeAgentTaskCount`) and end-to-end in `tests/mcp-subscribe-to-idle.test.mjs` via two fixtures ŌĆö `scenario-bg-task-hang.json` (turn_end while a subagent is live, never completes ŌåÆ wake deferred) and `scenario-bg-task-complete.json` (subagent completes + follow-up turn_end ŌåÆ single delivery).

Opt-in real-claude smoke (`smoke.real.test.mjs`, `RUN_REAL_CLAUDE=1`): spawns the actual CLI, asserts `system/init` + Ōēź1 `text_delta` + non-error `turn_end`. Cleans the session jsonl on exit.

The plugin system tests against **`tests/fixtures/fake-plugin/`** ŌĆö a dependency-free `node:http` child (health, relative-asset HTML, streaming `/echo`, hand-rolled WS echo, MCP endpoint with `echo`/`sleep`/`fail`/`transport-bug` tools pinning both halves of the child contract) plus `crash.mjs`/`slow-ready.mjs`/`forker.mjs` for the supervisor edge cases; `tests/plugin-helpers.mjs` clones it into temp projects roots. `bootServer` exposes `pluginHost` and stops plugin children on close.
