                Overage protection
                
          Action on overage
          
                
          
                  
                  
                  
                
          
                
          What to do the moment a session starts using paid overage credits. Stop soft-interrupts the turn (stays idle, manually resumable). Stop & resume also auto-resumes the session ~5s after the rate-limit window resets. Off by default.
          
                
                
          When on, the chosen overage action also fires once utilization crosses this percentage — before paid overage begins. Checked two ways: the live rate-limit stream (near Anthropic's own ~90% mark) and a periodic server-side usage poll of the 5-hour window, so low thresholds (e.g. 25%) trip even though the stream never reports that low. Independent of the paid-overage trip above. Has no effect when the action is Off. Off by default.

              Markdown guidance composed into Claude sessions, at three scopes that cascade
              Conductor → Workspace → Project. Each scope has an always-on
              core plus toggleable built-in conventions (read-only) and custom conventions you
              can add, edit, and delete.
            
            
            
            
              Workspace conventions
              
            
                Composed into the app-owned projects-root CLAUDE.md — the file every project
                imports via @../CLAUDE.md. Global; changes regenerate that file immediately
                (it is fully owned by code-conductor, overwritten on boot and on change).

              Conductor conventions
              
              
                Composed into the 🎼 Conduct orchestrator's guidance (.conduct/CONDUCT.md).
                Global — applies to every conductor session; changes regenerate the doc for the
                next-spawned conductor. Keep the enabled set lean — every convention costs tokens in every
                conductor session.
